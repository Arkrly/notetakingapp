## Integration Issues

### Critical (Blocking)

#### 1. Note Update Destroys Pin/Archive State
**From:** `NoteFormComponent` → `NoteServiceImpl.applyUpdate`  
**Issue:** When editing a note, the frontend only sends title, content, color, and tags. The backend's `applyUpdate()` method unconditionally overwrites `isPinned` and `isArchived` fields:

```java
// NoteServiceImpl.java:93-99
private void applyUpdate(Note note, UpdateNoteRequest request) {
    note.setTitle(request.getTitle());
    note.setContent(request.getContent());
    note.setIsPinned(request.getIsPinned() != null ? request.getIsPinned() : note.getIsPinned()); // ← BUG
    note.setIsArchived(request.getIsArchived() != null ? request.getIsArchived() : note.getIsArchived()); // ← BUG
    // ...
}
```

**Affected Requirements:** FEAT-02, FEAT-03  
**Severity:** 🔴 HIGH
