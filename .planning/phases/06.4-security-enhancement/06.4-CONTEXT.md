## Implementation Decisions

### Tag IDOR Analysis
- Tags are stored as a String field in Note entity (not a separate Tag entity)
- No separate TagRepository or TagService exists
- All tag access goes through Note endpoints
- Note endpoints already verify ownership via findNoteAndVerifyOwnership()

### Security Status
- SEC-02 verification from Phase 3 already confirmed: Tags are protected by Note ownership verification
- No separate tag endpoints exist that could have IDOR vulnerability
- User cannot access "other users' tags by manipulating tag IDs" because there are no tag IDs
