## Technical Context

### Current Implementation Issues

#### 1. Tag Normalization (FEAT-01)
**Problem:** No lowercase conversion or duplicate prevention across the stack
**Impact:** "Work" and "work" stored as separate tags

**Current Flow:**
- Frontend: Tags added as-is from user input
- Backend: Tags stored as String field in Note entity
- No normalization at any layer

**Required Changes:**
- Frontend: Convert tags to lowercase before sending to API
- Backend: Convert tags to lowercase on save
- Backend: Prevent duplicate tags (case-insensitive comparison)

#### 2. Pin/Archive Preservation (FEAT-02, FEAT-03)
**Problem:** `NoteServiceImpl.applyUpdate()` overwrites pin/archive status when not in request

**Current Code (NoteServiceImpl.java:93-99):**
```java
private void applyUpdate(Note note, UpdateNoteRequest request) {
    note.setTitle(request.getTitle());
    note.setContent(request.getContent());
    note.setIsPinned(request.getIsPinned() != null ? request.getIsPinned() : note.getIsPinned()); // ← BUG
    note.setIsArchived(request.getIsArchived() != null ? request.getIsArchived() : note.getIsArchived()); // ← BUG
    // ...
}
```

**Root Cause:** Frontend only sends title, content, color, tags in update request
**Impact:** Pin and archive status reset to false on every edit

**Required Changes:**
- Frontend: Include isPinned and isArchived in update request
- Backend: Only update fields that are present in request (null check already exists but logic is wrong)

#### 3. Archive Filtering (FEAT-04)
**Problem:** Backend returns all notes, frontend filters archived notes only

**Current Flow:**
- Backend: `NoteService.getAllNotes()` returns all notes (no archive filter)
- Frontend: `notes-list.component.ts:185` filters archived notes

**Required Changes:**
- Backend: Add `isArchived = false` filter to main query methods
- Backend: Add separate endpoint for archived notes if needed
- Frontend: Remove client-side filtering (backend handles it)

## Implementation Decisions

### Tag Normalization Strategy
- **Frontend:** Convert to lowercase before API call
- **Backend:** Convert to lowercase on save + deduplicate
- **Rationale:** Defense in depth - both layers normalize

### Pin/Archive Preservation Strategy
- **Frontend:** Always include isPinned and isArchived in update request
- **Backend:** Keep existing null check logic (it's correct)
- **Rationale:** Minimal backend change, fix is in frontend

### Archive Filtering Strategy
- **Backend:** Add `WHERE is_archived = false` to main queries
- **Frontend:** Remove client-side filtering
- **Rationale:** Single source of truth, reduces data transfer

// Backend (NoteServiceImpl.java)
private void normalizeTags(Note note) {
    Set<String> normalized = new HashSet<>();
    for (String tag : note.getTags()) {
        normalized.add(tag.toLowerCase());
    }
    note.setTags(new ArrayList<>(normalized));
}

## Validation Architecture

### Test Strategy
1. **Tag Normalization:**
   - Create note with tags "Work", "work", "WORK"
   - Verify only one tag stored (lowercase)
   - Verify duplicate prevention

2. **Pin Preservation:**

   - Edit note (change title)
   - Verify pin status preserved

3. **Archive Preservation:**
   - Create note, archive it
   - Edit note (change content)
   - Verify archive status preserved

4. **Archive Filtering:**
   - Create 3 notes, archive 1
   - Call getAllNotes()

   - Verify archived note not in results
