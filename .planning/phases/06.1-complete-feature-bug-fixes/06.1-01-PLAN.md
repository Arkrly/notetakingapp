# Plan 01: Tag Normalization

**Goal:** Tags are normalized to lowercase on save in both frontend and backend, preventing duplicate tags of different cases.

## Success Criteria
- Tags are converted to lowercase before sending to API
- Tags are converted to lowercase on save in backend
- Duplicate tags are prevented (case-insensitive comparison)
- "Work", "work", "WORK" all become "work" (single tag)
