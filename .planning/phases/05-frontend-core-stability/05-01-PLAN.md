<action>
Add takeUntilDestroyed() operator to the valueChanges subscription in ProfileComponent.

In profile.component.ts:
1. Add import: `import { takeUntilDestroyed } from '@angular/core/rxjs-interop';`
2. Find the subscription in ngOnInit (line 69): `this.passwordForm.get('newPassword')?.valueChanges.subscribe(val => {`
3. Modify to use pipe with takeUntilDestroyed:
```typescript
this.passwordForm.get('newPassword')?.valueChanges.pipe(
  takeUntilDestroyed()
).subscribe(val => {
  this.calculateStrength(val);
});
```

Why this subscription needs cleanup:
- It's created in ngOnInit and lives for the component lifetime
- Without cleanup, it continues emitting after component destruction
- Other subscriptions in the component are fire-and-forget (button clicks, form submits)

Do NOT modify:
- Other subscriptions (they are short-lived event handlers)
- Any other component logic
</action>
