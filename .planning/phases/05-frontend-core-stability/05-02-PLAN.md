<name>task 1: Verify 401 handling flow works correctly</name>
<files>none - manual verification</files>
<what-built>
The error interceptor (error.interceptor.ts) already handles 401 errors by:
- Calling authService.logout() to clear token and redirect
- Showing toast notification: "Session expired. Please login again."
- Redirecting to /login route

This is already implemented per the research findings. Need to verify it works end-to-end.
</what-built>
<how-to-verify>
Test the 401 handling flow:

1. Start the frontend dev server:
   ```bash
   cd notetakingapp-frontend
   npm start
   ```

2. Open browser to http://localhost:4200

3. Simulate an expired/invalid token:
   - Open browser DevTools console
   - Run: `localStorage.setItem('note_app_token', 'invalid_token_here')`

4. Navigate to http://localhost:4200/notes (or try to create/edit a note)

5. Expected behavior:
   - Toast notification appears: "Session expired. Please login again."
   - Token is cleared from localStorage
   - Browser redirects to /login page
   - Console shows "Session expired" error (from interceptor)

6. Verify token is cleared:
   - Check: `localStorage.getItem('note_app_token')` returns null

If all steps work as expected, the 401 handling is verified.
</how-to-verify>
<resume-signal>Type "approved" if 401 handling works correctly, or describe any issues encountered</resume-signal>
</task>
