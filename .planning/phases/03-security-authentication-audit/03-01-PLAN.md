# Plan 01: Fix Bulk Operations IDOR

## Objective

**Phase:** 03-security-authentication-audit  
**Goal:** Fix IDOR vulnerability in bulk delete operations  
**Purpose:** Prevent users from manipulating note IDs to access other users' data in bulk operations  

## Context

@.planning/phases/03-security-authentication-audit/03-CONTEXT.md  
@.planning/phases/03-security-authentication-audit/03-RESEARCH.md

### Current State

- Single-note operations (get/update/delete) use `findNoteAndVerifyOwnership()` - PROTECTED
- Bulk delete uses `deleteAllByIdInAndUserId()` - includes userId in query but NO count verification
- **Gap:** Bulk delete silently succeeds even if some note IDs don't belong to the user

### Expected Behavior (from CONTEXT.md)

Per user decision: "Fetch all notes by IDs and userId, verify count matches requested IDs"

### task 2: Add ownership verification to bulk delete

<task type="auto">
  <name>task 2: Verify ownership in bulk delete</name>
  <files>backend/src/main/java/com/backend/service/impl/NoteServiceImpl.java</files>
  <action>
Update bulkDeleteNotes method in NoteServiceImpl to:
1. First count notes owned by user that match the requested IDs
2. If count != noteIds.size(), throw UnauthorizedException
3. Only proceed to delete if verification passes

Code pattern:
```java
@Override
@Transactional
public void bulkDeleteNotes(List<UUID> noteIds, UUID userId) {
    log.info("Bulk deleting {} notes for user {}", noteIds.size(), userId);
    
    // Verify all requested IDs belong to user
    long ownedCount = noteRepository.countByIdInAndUserId(noteIds, userId);
    if (ownedCount != noteIds.size()) {
        throw new UnauthorizedException("Access denied: Some notes don't belong to you");
    }
    
    noteRepository.deleteAllByIdInAndUserId(noteIds, userId);
}
```
  </action>

  </verify>
  <done>Bulk delete now verifies ownership before deletion</done>
</task>
